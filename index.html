Android APK'da C/C++ (Native) Tespit Akış Şemaları

Apktool ile açılmış bir uygulamanın derinlemesine analizi. Raporunuzdan çıkartılmış 28 bağımsız tespit yöntemi.

1. Dosya Sistemi ve Hiyerarşi Analizleri

Yöntem 1: Standart Paylaşımlı Nesne (.so) Kontrolü

lib/ dizini ve ABI klasörleri mimari kodun en temel kanıtıdır.

graph TD A[apktool ile APK yı aç] --> B{Kök dizinde lib klasörü var mı?} B -- Evet --> C{İçinde ABI klasörleri var mı?
armeabi-v7a vs} C -- Evet --> D{İçinde .so uzantılı
dosya var mı?} D -- Evet --> E[C/C++ Native Kod KESİNLEŞTİ] B -- Hayır --> F[Bu yöntemle bulunamadı]

Yöntem 2: C++ STL (Standard Template Library) Varlığı

Modern C++ altyapısının zorunlu taşıyıcısı.

graph TD A[lib dizinini incele] --> B{Dizinler içinde libc++_shared.so
isimli dosya var mı?} B -- Evet --> C[Uygulama %100 Modern C++
özellikleri ile derlenmiş] B -- Hayır --> D[Standart C veya farklı kütüphane]

Yöntem 3: Gizlenmiş (Obfuscated) ELF Dosyaları

assets/ veya res/ klasörlerindeki sahte uzantılı (örn. .mp3, .dat) native dosyalar.

graph TD A[assets dizinini tara] --> B{Şüpheli dosyaları
Hex Editor ile aç} B --> C{Dosya başlığı x7fELF
sihirli baytları ile mi başlıyor?} C -- Evet --> D[Gizlenmiş C/C++ İkili Dosyası
TESPİT EDİLDİ] C -- Hayır --> E[Normal veri dosyası]
